## 2. `pyproject.toml` target shape

```toml
[build-system]
requires = ["setuptools>=68", "wheel"]
build-backend = "setuptools.build_meta"

[project]
name = "startup-radar"
version = "0.1.0"                         # setuptools-scm comes in Phase 4
description = "Personal startup discovery radar with Streamlit dashboard"
readme = "README.md"
requires-python = ">=3.10"
license = { text = "MIT" }
dependencies = [
    "pyyaml>=6.0",
    "requests>=2.31.0",
    "feedparser>=6.0.10",
    "beautifulsoup4>=4.12.0",
    "python-dateutil>=2.8.2",
    "pandas>=2.0.0",
    "streamlit>=1.30.0",
    "duckduckgo-search>=6.0.0",
    "python-docx>=1.1.0",
]

[project.optional-dependencies]
google = [
    "google-api-python-client>=2.100.0",
    "google-auth-httplib2>=0.2.0",
    "google-auth-oauthlib>=1.2.0",
]

[tool.uv]
dev-dependencies = [
    "pytest>=8.0",
    "pytest-cov>=4.0",
    "ruff>=0.5",
    "mypy>=1.10",
    "vcrpy>=6.0",
]

[tool.setuptools]
py-modules = [
    "main", "daily_run", "app", "deepdive",
    "database", "filters", "models",
    "config_loader", "connections",
]

# (existing [tool.ruff], [tool.mypy], [tool.pytest.ini_options] blocks unchanged)
```

### Why `[tool.setuptools] py-modules`

Without this, setuptools' auto-discovery sees a flat repo with no package and refuses to build. `py-modules` explicitly lists the loose top-level Python files, making the project installable today and giving `pip install -e .` a coherent answer. Phase 3 deletes this block when files move into `startup_radar/`.

### Why move dev deps from `[project.optional-dependencies]` to `[tool.uv]`

- `[project.optional-dependencies] dev` is for downstream consumers (`pip install package[dev]`).
- `[tool.uv] dev-dependencies` is for repo developers; `uv sync` installs them by default and `uv sync --no-dev` excludes them.
- Avoids the duplication that would otherwise happen between the two blocks.
- `google` extras stay under `[project.optional-dependencies]` because some users genuinely run without Gmail/Sheets.

## 3. `Makefile` target shape

```make
.PHONY: help install install-dev test lint format format-check typecheck ci serve run doctor clean

help:
	@grep -E '^[a-zA-Z_-]+:.*?## ' $(MAKEFILE_LIST) | awk 'BEGIN {FS=":.*?## "}; {printf "  \033[36m%-14s\033[0m %s\n", $$1, $$2}'

install:        ## Sync runtime + dev deps via uv (lockfile → .venv)
	uv sync --all-extras

install-dev: install  ## Alias for install (dev deps included by default)

test:           ## Run pytest
	uv run pytest

lint:           ## Run ruff check
	uv run ruff check .

format:         ## Auto-format with ruff
	uv run ruff format .

format-check:   ## Check formatting without writing
	uv run ruff format --check .

typecheck:      ## Run mypy on typed modules
	uv run mypy

ci: lint format-check typecheck test  ## Full local CI

serve:          ## Start the Streamlit dashboard
	uv run streamlit run app.py

run:            ## Run the discovery pipeline once
	uv run python main.py

doctor:         ## Quick environment check
	@uv run python --version
	@test -f config.yaml && echo "config.yaml: ok" || echo "config.yaml: MISSING (copy from config.example.yaml)"
	@test -f startup_radar.db && echo "DB exists" || echo "DB not yet created"

clean:
	rm -rf .pytest_cache .mypy_cache .ruff_cache build dist *.egg-info
	find . -type d -name __pycache__ -prune -exec rm -rf {} +
```

### Notes

- Drop `PYTHON ?= python3` / `PIP ?= ...` variables — `uv run` is the universal entrypoint now.
- `uv sync --all-extras` installs `google` extra by default. If a user wants the lean install they can run `uv sync` (no extras). Document in README.
- Every Make target now requires `uv` on PATH. Add a hard prerequisite check at the top of `install`? **No** — error message from `make` will be clear enough.
